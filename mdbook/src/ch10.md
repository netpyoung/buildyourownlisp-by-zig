## 0x00

- Q-Expressions: 인용된 표현식(quoted expression)을 의미하며, 표준 Lisp 메커니즘으로 계산되지 않는 Lisp 표현식의 한 유형
  - `( )`로 감싸는게 아닌 `{ }` 로 감쌈
- qexpr을 eval하면 그때서야 child 에서 꺼내옴


``` c
    // c
    "                                                    \
      number : /-?[0-9]+/ ;                              \
      symbol : \"list\" | \"head\" | \"tail\" | \"eval\" \
             | \"join\" | '+' | '-' | '*' | '/' ;        \
      sexpr  : '(' <expr>* ')' ;                         \
      qexpr  : '{' <expr>* '}' ;                         \
      expr   : <number> | <symbol> | <sexpr> | <qexpr> ; \
      lispy  : /^/ <expr>* /$/ ;                         \
    ",
```

``` zig
//    zig
    const lang =
        \\ number : /-?[0-9]+/ ;
        \\ symbol : "list" | "head" | "tail"
        \\        | "join" | "eval" | '+' | '-' | '*' | '/' ;
        \\ sexpr  : '(' <expr>* ')' ;
        \\ qexpr  : '{' <expr>* '}' ;
        \\ expr   : <number> | <symbol> | <sexpr> | <qexpr> ;
        \\ lispy  : /^/ <expr>* /$/ ;
    ;
```
